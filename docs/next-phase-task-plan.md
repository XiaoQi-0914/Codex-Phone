# Codex Phone 下一阶段任务驱动文档

## 目标

在当前可运行的单项目 MVP 基础上，补齐最关键的项目扩展能力、会话能力和使用体验，让它从“能用”变成“更顺手、更稳、更方便继续迭代”。

## 当前判断

现阶段已经完成了主链路：

- 手机网页可以连到电脑端服务。
- 能发送消息给本机 Codex。
- 能接收 JSONL 事件流并展示。
- 支持会话持久化和简单恢复。
- 支持运行后自动看 `git status` / `git diff --stat`。

因此下一阶段不该优先继续堆功能，而应该先补三类能力：

1. 多项目支持。
2. 会话与体验细化。
3. 运行可靠性。

## 优先级一：多项目支持

### 任务 1.1：把单项目配置升级为白名单列表

问题：当前写死单项目路径，后续项目一多就会卡死在这一步。

要做的事：

- 把 `project` 改成 `projects` 列表。
- 每个项目包含 `id`、`name`、`path`。
- 后端只允许从白名单中选择项目，不接受手机端传任意路径。

完成标准：

- 可以配置多个本地项目。
- 手机端只能切换白名单里的项目。
- 不同项目之间的会话互不串线。

### 任务 1.2：会话绑定项目

要做的事：

- 每个会话必须明确绑定一个项目。
- 会话列表按项目分组或在切换项目后只显示当前项目会话。
- 切换项目时自动切换对应会话上下文。

完成标准：

- 不会出现跨项目误操作。
- 切项目后历史会话仍可恢复查看。

## 优先级二：会话管理完善

### 任务 2.1：支持会话重命名

问题：当前会话标题只靠第一条消息自动生成，后期容易失真。

要做的事：

- 支持手动重命名会话。
- 保留自动标题作为默认值。
- 标题变更后立即持久化。

完成标准：

- 用户可以把“新会话”改成更有意义的名字。
- 刷新后标题不丢。

### 任务 2.2：支持会话搜索与筛选

要做的事：

- 按标题、项目名、时间筛选会话。
- 支持关键字搜索最近会话。

完成标准：

- 会话多了以后还能快速找到目标。
- 搜索结果能直接切换进入。

### 任务 2.3：补全会话生命周期

要做的事：

- 区分删除、归档、恢复。
- 删除前给出更明确的后果提示。
- 保留必要的历史审计记录。

完成标准：

- 用户不会因为误触丢失重要上下文。
- 删除和归档行为可预期。

## 优先级三：运行可靠性

### 任务 3.1：完善断线重连和状态恢复

问题：手机端断网、页面刷新、服务重启后，当前状态恢复还可以更稳。

要做的事：

- WebSocket 重连后自动同步当前会话、运行状态和最新消息。
- 发生进程异常时给出明确提示。
- 保留最后一次运行摘要。

完成标准：

- 页面刷新后能回到正确的会话和状态。
- 断线后不会出现前端和后端状态分裂。

### 任务 3.2：增加运行保护

要做的事：

- 给长时间无响应的 Codex run 增加超时保护。
- 明确标记取消、失败、超时三种结果。
- 运行中禁止触发冲突操作。

完成标准：

- 卡住的任务能被识别为超时。
- 用户能看懂当前任务到底是停了、错了还是完成了。

## 优先级四：手机端体验

### 任务 4.1：把消息展示做得更像“可读的工作流”

要做的事：

- 将命令、摘要、错误、普通输出分成更清晰的卡片。
- 对代码块、列表、标题做更完整的渲染。
- 对长输出默认折叠。

完成标准：

- 手机上扫一眼就能分辨哪些是命令、哪些是结果、哪些是警告。

### 任务 4.2：补一个更清晰的执行状态区

要做的事：

- 显示当前项目、当前会话、当前 run 状态。
- 显示最近一次退出码和摘要。

完成标准：

- 用户不需要翻聊天记录就能知道现在卡在哪一步。

## 以后再补

如果后面要把它从“单人工具”慢慢放大到“多人或更公开的工具”，再补访问控制、授权、来源限制这一类安全能力就很合适。现在先不把它放进主线。

## 建议实施顺序

1. 先做配对/授权。
2. 再把单项目改成白名单多项目。
3. 然后补会话重命名和搜索。
4. 接着做断线恢复和超时保护。
5. 最后打磨手机端展示。

## 这阶段的验收标准

如果下面这些都成立，就可以认为下一阶段完成得比较扎实：

- 未授权的人不能控制本机 Codex。
- 至少可以稳定管理多个本地项目。
- 会话可以命名、查找和恢复。
- 页面刷新和断线后状态能接回来。
- 手机上看输出不费劲，能快速判断执行结果。

## 2026-06-17 轻量收尾实施记录

本轮目标：一次性补齐剩余低成本体验项，不扩大成复杂产品功能。

准备做的改动：

- 会话重命名：新增 `rename_session` WebSocket 消息，后端更新 SQLite 会话标题，前端用轻量 `prompt` 触发。
- 会话搜索：侧边栏本地按当前项目会话标题过滤，不做跨项目全文搜索。
- 断线重连：前端 WebSocket 断开后每 2 秒自动重连，重连成功后重新同步 `hello` 状态。
- 运行超时提示：后端在 Codex run 超过 5 分钟后广播一次软提醒，不自动杀进程。
- 当前状态区：页面顶部下方增加紧凑状态栏，显示项目、会话、连接/运行状态和最近执行结果。
- UI 原则：保持手机端紧凑清晰，避免大块卡片挤占聊天空间。

不做的事：

- 不做登录/配对码。
- 不做复杂归档恢复。
- 不做跨项目全文检索。
- 不做自动 kill 超时任务。
- 不做大规模 UI 重构。
