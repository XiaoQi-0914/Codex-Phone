# Electron 托盘版实施清单

## 本次目标

把当前 Codex Phone 从“终端里运行的本地服务”升级成“Windows 可执行托盘应用”。

## 本次准备做的改动

1. 桌面壳子

- 引入 Electron。
- 增加 Electron 主进程入口。
- 提供主窗口和启动提示窗。

2. 托盘交互

- 启动后创建系统托盘图标。
- 托盘菜单支持打开、隐藏、复制访问地址、重启服务、退出。
- 关闭主窗口时默认隐藏到托盘。

3. 后端托管

- Electron 启动时自动拉起当前 Node 服务。
- Electron 退出时安全关闭服务。
- 支持托盘菜单里重启服务。

4. 启动提示

- 启动后弹出轻量提示窗。
- 显示局域网 IP、端口和访问地址。
- 提供复制访问地址按钮。

5. 构建与打包

- 增加 Electron 开发脚本。
- 增加桌面构建脚本。
- 增加 Windows 打包配置。

6. 文档与验证

- 更新 README 的启动方式。
- 记录桌面版运行和打包方式。
- 跑构建和基础验证直到通过。

## 本轮补充收尾

1. 图标资产入库

- 把桌面图标 SVG 放进项目目录，避免继续依赖桌面绝对路径。
- 生成多尺寸 PNG / ICO，保证托盘和安装包图标都稳定。

2. 启动失败体验补强

- 失败窗口内直接提供“复制完整错误信息”按钮。
- 明确展示安装目录、配置目录、数据目录和服务入口。
- 提供一键打开这些目录的按钮。

3. 托盘与提示补充

- 启动提示里补上端口展示。
- 托盘菜单里加入“打开安装目录”。

4. 最终验证

- 重新跑图标构建、前后端构建和桌面打包。
- 确认产物时间戳更新，并验证 release 目录可正常生成。

## 本轮修复

1. 安装包后端启动失败

- 修正打包后 Electron 主进程定位后端入口的路径。
- 区分 `app.asar` 内代码目录和安装目录 / resources 目录。
- 确保服务端静态资源继续指向打包后的 `dist/web`。

2. 安装路径可控

- 把 NSIS 安装器改成可选择安装目录。
- README 补充默认路径和卸载清理说明。

3. 重新验证

- 重新跑构建和安装包。
- 用 `win-unpacked` 实际启动，确认 5179 能起来。

## 本轮方向修正

1. 交互目标收敛

- 去掉“本机前端主窗口”这个桌面 App 思路。
- 改成“纯后台服务启动器 + 托盘菜单 + 浏览器打开访问页”。

2. 托盘行为调整

- 托盘双击直接打开默认浏览器访问页。
- 菜单保留复制地址、查看服务信息、打开配置/数据/安装目录、重启、退出。

3. 保留手机访问所需资源

- 仍然把网页静态资源一起打包到服务里。
- 这是为了让手机访问 `5179` 时还能看到聊天页，不是为了做桌面前端。
